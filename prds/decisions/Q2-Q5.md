# Decision log — Q2 (Manim version) & Q5 (local vs container)

Decided in M0 (Phase 0). These are pinned and referenced everywhere downstream.

## Q2 — Which Manim CE version do we pin?

**Decision: Manim Community Edition `v0.18.1`.**

- Pinned once in `core/config.py` (`manim_version`, `manim_base_image`) and in the `Dockerfile` base tag — nowhere else hardcodes a version.
- Rationale: Manim CE's API drifts between releases; training data mixes CE versions and ManimGL, which silently breaks generated code. A single pinned version lets the M1 codegen prompt (`prompts/manim-conventions.md`) target a fixed API surface.
- To change it later: bump the tag in `core/config.py` + `Dockerfile`, rebuild (`python -m scripts.setup_env`), and re-run the M3 eval battery to catch regressions before committing.

## Q5 — Local single-process render, or containerized from the start?

**Decision: containerized from day one**, using the official `manimcommunity/manim` image.

- Rationale:
  1. The toolchain (Manim + LaTeX + FFmpeg) is the real Phase-0 risk. The official image bundles all three, working — it removes the entire host-install problem (this machine had **no** FFmpeg, LaTeX, or Manim).
  2. The container gives the **no-network / non-root sandbox for free** (`--network=none`, image user `manimuser`) — exactly what FR-23 needs, at zero extra cost now. M7 only has to *harden* it (resource caps, ephemeral FS), not invent it.
  3. Reproducible: one command (`python -m scripts.setup_env`) builds an identical env on any machine with Docker.
- Trade-off accepted: a ~2 GB base-image pull on first setup, and Docker Desktop must be running. Worth it vs. fighting a Windows LaTeX/FFmpeg install.
- The host side stays tiny: pure-Python orchestration (`render/sandbox.py`) shells out to `docker run`. No Manim/LaTeX/FFmpeg on the host.

### Consequence for the build
- `render/sandbox.py::build_command` is the single chokepoint where code meets the renderer — every render (sample now, model-written code later) goes through it with `--network=none` by default.
- Other open questions (Q1 model+budget, Q3 TTS, Q4 scene cap, Q6 canonical workflow) are deferred to the milestones that need them (M2/M4/M5), per `MILESTONES.md`.
